## Package Overview

`imageTCGAWorkflow` is a **Bioconductor workflow package** (`BiocType: Workflow`) that wraps four packages in the imageTCGA ecosystem into a single end-to-end vignette covering TCGA histopathology image discovery, feature retrieval, spatial analysis, and visualization.

**Ecosystem packages (all GitHub-only during development):**
| Package | GitHub | Role |
|---|---|---|
| `imageTCGA` | `billila/imageTCGA` | Shiny app for image discovery & download-code generation |
| `imageFeatureTCGA` | `waldronlab/imageFeatureTCGA` | Import HoVerNet / Prov-GigaPath features → SpatialExperiment / MAE |
| `imageTCGAutils` | `waldronlab/imageTCGAutils` | PCA, Moran's I, Geary's C, LISA on tile/nucleus data |
| `HistoImagePlot` | `waldronlab/HistoImagePlot` | Overlay cell segmentation on tissue thumbnails |

## Common Commands

```r
# Install ecosystem dependencies from GitHub
remotes::install_github("billila/imageTCGA")
remotes::install_github("waldronlab/imageFeatureTCGA")
remotes::install_github("waldronlab/imageTCGAutils")
remotes::install_github("waldronlab/HistoImagePlot")

# Development
devtools::load_all()
devtools::document()    # regenerate NAMESPACE + man/ from roxygen2
devtools::check()       # R CMD CHECK
devtools::build_vignettes()

# Build pkgdown website locally
pkgdown::build_site()
```


## Architecture Notes

- The **vignette is the product**. This is a workflow package — the R/ directory contains
  only a package-level doc file. All scientific content lives in `vignettes/imageTCGAWorkflow.Rmd`.
- The vignette has `cache = TRUE` globally because several steps hit remote APIs
  (imageFeatureTCGA data catalogue, BiocFileCache downloads).
- The pkgdown site is deployed to `gh-pages` branch and will be served at
  `https://billila.github.io/imageTCGAWorkflow`. Enable GitHub Pages (source: `gh-pages` branch)
  in the repository Settings after the first push.
- The GitHub Actions workflows install the four ecosystem packages from GitHub before
  running check or building the site, since they are not yet on Bioconductor/CRAN.

## Bioconductor Workflow Conventions

- `DESCRIPTION` must have `BiocType: Workflow` and `biocViews` including `Workflow`.
- Documentation is generated by roxygen2; run `devtools::document()` after any `#'` changes.
- `NAMESPACE` is auto-generated — do not edit manually.
- `LazyData: false` is required for Bioconductor submission.
